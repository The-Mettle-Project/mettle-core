A native systems language

Straight to the metal, and it shows its work.

Mettle compiles to native x86-64 through its own backend, linker, and debugger. The auto-vectorizer beats gcc -O3 on data-parallel kernels, and the compiler explains every optimization it makes. No LLVM. No VM.

// the toolchain

A whole compiler, built from scratch.

No LLVM, no VM, no external assembler. Every part of the path from source to running binary is Mettle's own.

01
01

Native backend & linker

Emits x86-64 object code directly and links it with a built-in PE linker on Windows, ELF on Linux. No assembler, no toolchain dependency.

02
02

AVX2 auto-vectorizer

Maps, reductions, dot products, and search loops become packed SIMD, bit-exact for integers. @simd! makes it a contract.

03
03

Source-level debugger

Breakpoints, stepping, call stacks, live variable read and write. No gdb, no DWARF. Faults stop at the source line, variables intact.

04
04

Compile-time memory safety

Use-after-free, double-free, leaks, and null dereferences caught before you run, interprocedurally, with concrete fixes.

05
05

Optimization contracts

@simd!, @inline!, @noalloc: declare intent and the build proves it or fails, pointing at the reason.

06
06

GPU offload

Write kernel functions, launch with dispatch, and Mettle emits NVIDIA PTX through a native CUDA path. No nvcc.
